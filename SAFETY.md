# 安全使用指南

## ⚠️ 重要说明

本项目使用逆向工程的 X (Twitter) API 进行数据收集。虽然功能完整且免费，但存在一定风险。

## 🛡️ 已实施的安全措施

### 1. **频率控制**
- ✅ 请求之间随机延迟 3-8 秒
- ✅ 模拟人类行为模式
- ✅ 出错后等待 5 秒再继续
- ✅ 每日只抓取高优先级创作者（默认 10 位）

### 2. **请求限制**
- ✅ 默认只抓取 `priority: "high"` 的创作者
- ✅ 如果没有高优先级，最多抓取 5 位
- ✅ 60 秒超时保护
- ✅ 错误自动跳过，不会重试

### 3. **数据最小化**
- ✅ 只收集必要字段
- ✅ 只保留今天的推文
- ✅ 临时文件自动清理

## 📊 风险评估

### 低风险行为（推荐）✅
- 每天运行一次
- 只抓取 5-10 位创作者
- 使用随机延迟
- 避免高峰时段

### 中风险行为（谨慎）⚠️
- 每天运行多次
- 抓取 20+ 创作者
- 固定时间运行

### 高风险行为（不推荐）❌
- 频繁运行（每小时）
- 大量并发请求
- 无延迟连续请求
- 使用多个 IP 地址

## 🔧 配置建议

### creators.json 优先级设置

只将最重要的创作者设为 `high` 优先级：

```json
{
  "username": "karpathy",
  "priority": "high",  // 每日抓取
  "active": true
}
```

```json
{
  "username": "lessImportant",
  "priority": "medium",  // 不会被每日抓取
  "active": true
}
```

### 建议的抓取策略

**日常使用（推荐）：**
- 5-10 位高优先级创作者
- 每天早上 08:00 运行
- 随机延迟 3-8 秒

**保守策略（最安全）：**
- 3-5 位创作者
- 每天运行
- 延迟 5-10 秒

**激进策略（有风险）：**
- 15+ 创作者
- 每天多次
- 不推荐

## 🚨 警告信号

如果遇到以下情况，说明可能被检测：

1. **请求被拒绝**
   - 频繁 429 错误
   - 连接超时增加

2. **数据质量下降**
   - 返回空结果
   - 数据不完整

3. **账号异常**
   - 需要验证码
   - 登录失效

**遇到警告信号时的应对：**
- 立即停止抓取 24 小时
- 减少创作者数量
- 增加延迟时间
- 检查网络环境

## 🔄 备份方案

如果逆向 API 失效或被封：

- 成本：$100/月起

### 方案 B：RSS Feeds
- 某些创作者提供 RSS
- 功能有限但稳定

### 方案 C：手动整理
- 暂停自动化
- 手动选择重要内容

## 📝 最佳实践

1. **定期检查**
   - 每周查看脚本运行状态
   - 监控成功率
   - 调整创作者列表

2. **控制规模**
   - 从小规模开始（3-5 位）
   - 逐步增加
   - 观察稳定性

3. **时间分散**
   - 避免在固定时间运行
   - 可以随机化运行时间（如 07:00-09:00 之间）

4. **保持低调**
   - 个人使用为主
   - 不公开宣传抓取行为
   - 不商业化使用数据
